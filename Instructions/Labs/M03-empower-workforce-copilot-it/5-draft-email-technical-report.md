
# Rédiger un e-mail basé sur un rapport technique à l’aide de Copilot dans Outlook
---
Copilot dans Outlook est un outil basé sur l’IA qui peut aider les professionnels de l’informatique à épargner du temps et du travail lors de la rédaction d’e-mails personnalisés et de réponses par e-mail. Avec Copilot, vous pouvez générer rapidement des brouillons d’e-mail que Copilot adapte à vos besoins spécifiques, sans avoir à passer des heures à les écrire et à les modifier. Copilot peut analyser le style et le contenu de ce que vous écrivez, puis suggérer des expressions et des phrases pertinentes pour l’e-mail que vous rédigez. Cette fonctionnalité peut vous épargner du temps et du travail en réduisant le temps que vous passez à réfléchir à ce qu’il faut écrire.

Dans cet exercice, vous utilisez Copilot dans Outlook pour rédiger un nouvel e-mail basé sur le rapport que vous avez mis à jour dans l’exercice précédent.

> [!IMPORTANT]
> Les scénarios Copilot dans Outlook sont disponibles seulement sur la boîte aux lettres principale d’un utilisateur. Ils ne sont pas disponibles sur la boîte aux lettres d’archivage d’un utilisateur, sur les boîtes aux lettres de groupe, ou sur les boîtes aux lettres partagées et déléguées auxquelles l’utilisateur a accès. Microsoft prend en charge Copilot dans Outlook seulement pour les boîtes aux lettres hébergées sur Exchange Online. En outre, Copilot dans Outlook est pris en charge seulement dans les comptes professionnels ou scolaires Microsoft 365, et dans les comptes Microsoft avec des domaines de courrier spécifiques. Il est pris en charge via Copilot pour Microsoft 365 avec votre compte Microsoft Entra ID et via Copilot Pro avec votre compte MSA. Tout compte Microsoft utilisant un compte provenant d’un fournisseur de messagerie tiers peut néanmoins utiliser Outlook, mais il n’a pas accès aux fonctionnalités Copilot dans Outlook.

Copilot peut créer plusieurs brouillons du même e-mail, chacun d’une longueur et d’un ton différents. Vous pouvez ensuite parcourir les différents brouillons en sélectionnant les flèches avant et arrière. Copilot vous indique dans quel brouillon vous êtes, par exemple 2 sur 5. Vous pouvez sélectionner la flèche avant ou arrière pour revenir rapidement à une version particulière.

Quand vous passez en revue un brouillon généré, gardez à l’esprit les meilleures pratiques suivantes que vous avez apprises lors d’une formation précédente :

 -  Notez la formule d’appel au début (Cher(e)/Bonjour/Salut, etc.) et la formule de politesse à la fin (Sincèrement/Cordialement/Merci, etc.). Sont-elles dans le ton approprié pour le public cible et pour le type de message que vous voulez envoyer ?
 -  Notez le ton global du message. Est-il trop informel, trop formel ou trop direct pour le type d’e-mail que vous voulez ?
 -  Notez la longueur du message. Est-il trop court, sans expliquer les choses d’une façon suffisamment détaillée ? Ou est-ce qu’il est trop long, expliquant les choses avec trop de détails ? Et la longueur est-elle appropriée pour le public cible ?
 -  Le message explique-t-il les éléments dans des termes trop simplistes ou trop compliqués pour le public cible ?

### Exercice

Dans l’exercice précédent, vous avez pris le rôle de consultant principal de Trey Research, qui est responsable de la gestion de l’installation d’un VPN dans le réseau d’entreprise de Fabrikam. Dans cet exercice, vous avez téléchargé un rapport intitulé **Trey Research - VPN Technical Overview.docx**, que vous avez ensuite mis à jour en utilisant Copilot dans Word.

Dans cet exercice, vous continuez avec ce rôle. Avec votre rapport maintenant finalisé, vous êtes prêt à planifier une réunion avec le personnel informatique de Fabrikam pour présenter les résultats du rapport. Fabrikam est nouveau dans le monde des VPN : vous prévoyez donc de profiter de cette réunion pour les informer sur les VPN avant d’en commencer l’installation. Vous prévoyez d’utiliser Copilot dans Outlook pour créer un e-mail à destination du personnel informatique de Fabrikam pour discuter de cette réunion à venir. Vous voulez aussi utiliser l’e-mail pour préparer le personnel à la réunion en résumant les résultats du rapport. Procédez comme suit pour créer l’e-mail en utilisant Copilot dans Outlook :

1.  Dans **Microsoft 365**, ouvrez **Outlook**.
2.  Ouvrez un **Nouvel** e-mail.
3.  Sous l’onglet **Message** qui s’ouvre, sélectionnez **Copilot** dans le ruban. Dans le menu déroulant qui s’affiche, sélectionnez **Brouillon avec Copilot**.
4.  Dans le corps du message, une fenêtre **Brouillon avec Copilot** s’affiche. Sous le message **Que voulez-vous que cet e-mail dise ?**, vous voyez une icône **Options de génération**. Sélectionnez cette icône pour voir les options de Copilot lors de la création d’un e-mail.
5.  Dans le menu déroulant qui s’affiche, vous voyez que vous pouvez changer le ton et la longueur de l’e-mail. Vous décidez de commencer en définissant le **Ton** sur **Formel** et la **Longueur** sur **Moyenne**.
6.  Entrez le prompt suivant dans le champ **Que voulez-vous que cet e-mail dise ?**. Ne sélectionnez pas encore le bouton **Générer**, car vous devez d’abord joindre le fichier **Trey Research - VPN Technical Overview.docx** à l’e-mail :
    
    **Je suis consultant principal pour Trey Research. Je vais gérer l’implémentation d’un VPN dans le réseau d’entreprise de Fabrikam. Faites le brouillon d’un e-mail au personnel informatique de Fabrikam, indiquant que je voudrais les rencontrer pour leur présenter les VPN et expliquer comment nous allons en implémenter un dans leur réseau. La majeur partie de la réunion sera basée sur le rapport en pièce jointe, qui donne une vue d’ensemble des VPN. Dites-leur qu’une demande de réunion leur sera adressée prochainement.** 
7.  Après avoir entré ou collé le prompt, sélectionnez l’option **Joindre un fichier** dans le ruban **Message**. Dans le menu qui s’affiche, le fichier **Trey Research - VPN Technical Overview.docx** doit apparaître dans la liste des **Fichiers suggérés**. Sélectionnez ce fichier. Au bout de quelques secondes, le nom du fichier doit apparaître au-dessus du brouillon.
8.  Sélectionnez le bouton **Générer** qui apparaît dans le champ de prompt.
9.  Passez en revue le brouillon généré par Copilot. Même si vous êtes satisfait de celui-ci, pour cet exercice, vous allez générer plusieurs brouillons Copilot afin de voir des exemples de la façon dont Copilot modifie le contenu en fonction des modifications apportées au ton et à la longueur.
10. En haut du message, Copilot affiche le début du prompt que vous venez d’entrer (« **Je suis consultant principal pour Trey R...** »). Sélectionnez cette section du prompt pour voir ce qui se passe.
11. Copilot affiche une fenêtre **Réécriture avec Copilot** qui vous permet de modifier le prompt précédent et de régénérer un brouillon entièrement nouveau. À ce stade, vous êtes satisfait du message, mais vous voulez voir comment il changerait avec un ton et une longueur différents. Sélectionnez l’icône **Options de génération** qui apparaît sous le prompt. Changez le **Ton** en **Direct** et la **Longueur** en **Longue**, puis sélectionnez le bouton **Générer**.
12. Passez en revue le brouillon de Copilot. En haut de la fenêtre **Réécriture avec Copilot**, notez le lien « **Je suis consultant principal pour Trey R...** ». Notez également comment Copilot indique que ce brouillon est « **2 sur 2** », ce qui signifie que vous examinez le deuxième des deux brouillons Copilot. Vous pouvez sélectionner la flèche arrière (&lt;) pour revenir au brouillon Copilot précédent, qui est dans ce cas le premier brouillon avec un ton **Formel**. Si vous générez plusieurs brouillons Copilot, comme vous le faites dans cet exercice, vous pouvez utiliser ces flèches pour passer d’un brouillon à l’autre afin de trouver celui qui vous convient. Si vous trouvez un brouillon qui vous convient, vous pouvez sélectionner un des boutons disponibles pour le **Conserver** ou pour **Régénérer** un nouveau brouillon avec le même ton et la même longueur. Notez la formule d’appel et la formule de politesse, et le ton global du message pour ce brouillon avec un ton Direct. Comme l’e-mail n’était pas très long au départ, le changement de longueur n’a pas semblé avoir beaucoup d’effet. Cependant, vous n’aimez pas du tout le ton **Direct** et vous voulez donc générer un nouveau brouillon avec un ton différent.
13. Sélectionnez le prompt en lien qui se trouve en haut de la fenêtre (« **Je suis consultant principal pour Trey R...** ») pour revenir à la fenêtre **Réécriture avec Copilot**. Sélectionnez l’icône **Options de génération** et cette fois, changez le **Ton** en **Neutre** et la **Longueur** en **Longue**. Vous voulez que la **Longueur** soit identique à celle du brouillon **Formel**. De cette façon, quand vous comparez les brouillons **Formel** et **Neutre**, la longueur de chaque brouillon est la même. Ainsi, les différences entre les deux brouillons sont dues au **Ton**. Sélectionnez **Générer**.
14. Vérifiez la réponse. Notez la formule d’appel et la formule de politesse, et le ton global du message. Vous n’aimez pas le ton **Neutre**, qui ne vous semble pas assez professionnel pour ce type d’e-mail. Sélectionnez le prompt en lien qui se trouve en haut de la fenêtre (« **Je suis consultant principal pour Trey R...** ») pour revenir à la fenêtre **Réécriture avec Copilot**. Sélectionnez l’icône **Options de génération** et cette fois, changez le **Ton** en **Informel**. Sélectionnez **Générer**.
15. Vérifiez la réponse. Notez la formule d’appel et la formule de politesse, et le ton global du message. À ce stade, vous vous rendez compte que vous n’aimez pas les tons **Direct** ou **Neutre**, mais vous ne savez pas si vous préférez l’option **Formel** ou **Informel**. Si vous ne vous souvenez pas de la façon dont les deux messages sont apparus, prenez note de la formule d’appel et de la formule de politesse **Informelles** ainsi que de la qualité du message. Sélectionnez ensuite la flèche arrière jusqu’à ce que vous reveniez au premier brouillon Copilot, qui avait le ton **Formel**. C’est à vous de décider si vous voulez utiliser le brouillon **Informel** ou le brouillon **Formel**. Vérifiez simplement à ce que la version actuelle que vous consultez soit bien celle que vous avez choisie.
16. Une fois que vous êtes satisfait du ton et de la longueur d’un message, il se peut que vous ne soyez toujours pas satisfait du texte du message lui-même. Si vous ne voulez pas modifier le ton ou la longueur, vous pouvez néanmoins demander à Copilot de régénérer un nouveau message en utilisant le ton et la longueur souhaités sans demander d’autres modifications. Vous pouvez aussi entrer un prompt qui demande à Copilot d’apporter une modification au brouillon existant.
17. Dans ce cas, une fois que vous avez sélectionné le brouillon **Informel** ou le brouillon **Formel**, vous décidez de demander à Copilot d’apporter une modification au corps du message. Vous voulez que Copilot mentionne l’importance d’examiner le rapport joint avant la réunion et qu’il demande également au personnel d’être très attentif à la section sur les **meilleures pratiques en matière d’implémentation**. Dans le champ de prompt qui indique actuellement « **Souhaitez-vous modifier quelque chose ?** », entrez le prompt suivant, puis sélectionnez l’icône de flèche **Envoyer** à l’extrémité du champ.
    
    **Ajoutez une note sur l’importance pour eux d’examiner le rapport avant la réunion. Faites-leur savoir que votre présentation lors de la réunion va supposer qu’ils l’ont fait et qu’ils sont familiarisés avec les informations de base fournies dans le rapport. Demandez-leur également d’accorder une attention particulière à la section sur les meilleures pratiques en matière d’implémentation.**
    
    > [!WARNING]
    > Ne sélectionnez pas le bouton **Régénérer**. Cette action ne va pas mettre à jour le brouillon actuel en fonction de la demande faite dans le prompt. Au lieu de cela, si vous sélectionnez **Régénérer**, un message entièrement nouveau est généré, basé SEULEMENT sur ce prompt, ce qui n’est pas ce que vous voulez. Par conséquent, quand vous demandez une modification, veillez à sélectionner la flèche **Envoyer** à la fin du champ de prompt.
18. Si vous recevez un message d’erreur indiquant « **Désolé, un problème est survenu. Copilot y travaille.** », renvoyez le prompt. S’il échoue une deuxième fois, c’est qu’il s’agit d’un scénario où Copilot ne peut pas gérer correctement un prompt de suivi qui lui demande de faire plusieurs choses. Dans votre prompt initial, vous pouvez demander à Copilot d’effectuer plusieurs tâches, mais sur les prompts suivants, Copilot retourne souvent cette erreur quand vous lui demandez de faire plusieurs choses. Pour contourner cette situation, vous devez décomposer le prompt précédent en plusieurs prompts, chacun lui demandant d’effectuer une seule tâche. Si vous avez du temps et que vous voulez faire cette expérience, divisez le prompt précédent en trois prompts :
     -  **Ajoutez une note indiquant qu’il est important qu’ils examinent le rapport avant la réunion**.
     -  **Ajoutez une note leur indiquant que cette présentation suppose qu’ils soient familiarisés avec les informations de base fournies dans le rapport**.
     -  **Ajoutez une note leur demandant d’accorder une attention particulière à la section sur les meilleures pratiques en matière d’implémentation**.
19. Parcourez le brouillon mis à jour pour examiner les modifications. Après avoir examiné cette dernière itération, vous êtes satisfait du brouillon : sélectionnez donc le bouton **Conserver**.
20. Notez que le message s’affiche dans le corps de l’e-mail et que la fenêtre **Réécriture avec Copilot** disparaît. À ce stade, vous pouvez réviser manuellement l’e-mail et y apporter les modifications souhaitées.
21. Comme vous n’envoyez pas ce message, sélectionnez l’icône de corbeille (**Abandonner**) dans le coin supérieur droit de l’écran de l’e-mail pour supprimer ce brouillon.
